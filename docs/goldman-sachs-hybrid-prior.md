# Goldman Sachs World Cup 2026 prior 与盘口量化框架

本文档迁移自一段关于 2026 世界杯预测、Elo、Poisson 进球模型、Monte Carlo 赛事模拟和盘口量化策略的讨论。它在本项目中的定位不是复现 Goldman Sachs 的完整模型，而是把公开流出的 Goldman Sachs Global Investment Research 报告输出结果作为一个可引用、可版本化的外部基本面概率先验。

报告标题为 `World Cup 2026: Predictions, Probabilities, and Paths to Victory`，日期为 `2026-05-29`。公开流出的 PDF 副本显示为 Goldman Sachs Global Investment Research 报告，但带有研究报告使用限制标记，因此在项目中应写成“外部流出报告副本”或“Goldman Sachs report copy”，不要写成高盛官方公开网页。

## 1. 对网上中文信息图的可信度判断

网上流传的中文图大概率不是 Goldman Sachs 原版图，而是根据报告结论二次整理的信息图，甚至可能经过 AI 重绘。它的核心概率数字与报告相符，例如 Spain 26%、France 19%、Argentina 14%、Brazil 8%、England 约 5%。但图本身存在几个明显瑕疵：日期可能写成 `2025-05-29` 而不是报告日期 `2026-05-29`；2026 世界杯是 48 队赛制，淘汰赛先有 Round of 32，而图中若写成“1/8 决赛”则属于赛制标注错误；图把模型简化成“Elo + 近 20,000 场数据”，但报告实际是以 Elo 为核心解释变量的 Poisson 进球回归，再外接 Monte Carlo 赛事模拟。

两个常被附带引用的 GitHub 仓库 `ddm7018/Elo` 和 `ModelOriented/EloML` 不能作为该图来自高盛的证据。前者是通用 Elo 排名实现，后者是用于机器学习模型排序的 R 包。它们只能说明 Elo 可以开源实现，不能说明图里的预测由这些代码跑出，更不能说明它们是 Goldman Sachs 的模型代码。

因此，本项目采用的判断是：核心报告结论可以引用；中文信息图不能当作原始来源；GitHub 仓库不能当作高盛模型背书。

## 2. Elo 不是 Monte Carlo，也不是典型机器学习模型

Elo 本质上是一套动态评分系统。每支球队有一个当前实力分，比赛之后根据实际结果相对于赛前预期的偏差更新评分。强队赢弱队，涨分很少；弱队赢强队，涨分很多。它最早用于国际象棋，后来被移植到足球、电竞、在线竞技和模型排序等场景。

常见单场预期得分公式为：

```math
E_A = \frac{1}{1 + 10^{(R_B - R_A)/400}}
```

其中 `R_A` 和 `R_B` 分别为 A、B 两队 Elo 分，`E_A` 是 A 队赛前预期得分。实际结果通常写成：

```math
S_A =
\begin{cases}
1, & A \text{ win} \\
0.5, & \text{draw} \\
0, & A \text{ lose}
\end{cases}
```

赛后更新为：

```math
R_A' = R_A + K(S_A - E_A)
```

其中 `K` 是更新系数。这个公式的核心不是简单地赢了加分、输了扣分，而是奖励“高于预期”的表现，惩罚“低于预期”的表现。

从严格意义上说，Elo 不是通常意义上的机器学习模型。它没有复杂特征，没有神经网络训练过程，也没有从数据中自动学习复杂模式。广义上可以说它是一个极简在线学习或递推状态估计系统；狭义上，它更像统计评分规则或半经验评分模型。

Monte Carlo 是另一层。Elo 只负责估计球队强度或单场胜率输入；Monte Carlo 负责把单场不确定性放入完整世界杯赛程，反复模拟整届赛事，最后统计夺冠频率。二者关系可以写成：

```text
历史比赛结果 -> Elo 评分 -> 单场强度差 / 进球期望 -> Monte Carlo 赛事模拟 -> 晋级概率 / 夺冠概率
```

用工程建模语言说，Elo 是状态估计器，Monte Carlo 是不确定性传播器。

## 3. 为什么使用 Poisson 进球模型

足球进球数是典型低频计数变量：固定 90 分钟窗口内，一支球队可能进 0、1、2、3 球，进球事件稀疏，数量通常较小。Poisson 分布正适合这类“固定时间窗口内事件发生次数”的建模。

Poisson 分布为：

```math
P(X=k)=\frac{\lambda^k e^{-\lambda}}{k!}
```

其中 `X` 为进球数，`k` 为具体进球数，`lambda` 为该队在这场比赛中的期望进球数。

在足球预测中，常见做法不是直接预测谁赢，而是先估计双方进球期望：

```math
G_A \sim \text{Poisson}(\lambda_A)
```

```math
G_B \sim \text{Poisson}(\lambda_B)
```

再由比分矩阵推出胜、平、负概率。若假设两队进球独立，则：

```math
P(G_A=a,G_B=b)=P(G_A=a)P(G_B=b)
```

`a>b` 为 A 胜，`a=b` 为平，`a<b` 为 A 负。

Poisson 在本项目里的价值不只是胜负预测，而是盘口适配。它可以把基本面强弱差转换成比分分布，再进一步得到胜平负概率、让球覆盖概率、大小球概率和正确比分概率。

需要注意，简单 Poisson 模型有很强的简化假设。它默认进球事件相对独立，默认一支队的进球数主要由一个平均强度控制。但真实足球中，红牌、领先后的战术收缩、落后后的冒险压上、淘汰赛加时和点球都会破坏这个假设。更复杂的足球模型可能使用 Dixon-Coles 修正、双变量 Poisson、负二项分布或 xG 模型。但对宏观赛事路径模拟而言，Poisson 的优势是参数少、可解释、容易和 Elo / 回归 / Monte Carlo 拼接。

## 4. Goldman Sachs 报告中的算法链条

根据报告说明，Goldman Sachs 使用自 1978 年以来接近 20,000 场正式国际比赛数据，估计某队面对某对手时的进球数。模型的核心变量是赛前 Elo 评分差，同时加入进攻天赋、近期动量、心理或历史因素、地理因素等变量。进球数采用 Poisson 分布建模，然后通过 50,000 次 Monte Carlo 模拟跑完整届 2026 世界杯赛程。

可概括为：

```text
1978 年以来正式国际比赛
  -> Elo 差、进攻天赋、近期动量、心理/历史变量、地理变量
  -> Poisson 进球回归
  -> 单场比分分布
  -> 小组赛与淘汰赛递归模拟
  -> 50,000 次 Monte Carlo
  -> 晋级概率与冠军概率
```

报告里提到的变量包括：

- Elo：作为球队强度的主要变量。
- 进攻天赋：用国家队球员在顶级俱乐部赛事或国家联赛中的射手表现构造。
- 近期动量：本队最近若干场正式比赛进球数、对手最近若干场正式比赛失球数。
- 心理或历史因素：卫冕冠军、首次参赛球队、面对欧洲球队、传统强国等历史偏差。
- 地理因素：主场效应、首都到比赛地点距离、海拔、参赛国平均气温与比赛地气温差异。

报告还说明模型局限：非进攻型人才、门将点球能力、中场控制、边路深度、健康状况、个体球员状态、教练经验等因素无法充分建模；同时，将两队进球分布视为相互独立也会带来误差。报告对世界杯比赛净胜球预测的历史相关性约为 49%，2018 年约 43%，2022 年约 45%。这说明它比拍脑袋强，但不是预测机器。

## 5. 本项目不复现高盛模型，而是引用其预测结果

本项目没有必要重造 Goldman Sachs 的 Elo-Poisson-Monte Carlo 模型。高盛的数据清洗、球员变量、地理变量、历史比赛库和赛程模拟细节都远超个人项目可获得范围。强行复现不仅投入产出比低，还可能造成“伪复现”的误导。

正确定位是：Goldman Sachs 报告输出结果作为外部基本面概率先验。

```math
P_{GS}(team, stage)
```

这些概率代表机构研究模型压缩后的基本面判断。本项目自己的价值不在于重新估计基本面概率，而在于把该先验与博彩公司赔率、交易所价格和 Polymarket 概率进行比较，寻找市场定价偏差。

```math
Edge_i = P_{GS,i} - P_{Market,i}
```

```math
EV_i = P_{GS,i} \times Odds_i - 1
```

其中 `Odds_i` 为十进制赔率。若 `EV_i > 0`，则按照 Goldman Sachs 先验概率计算存在正期望；若 `EV_i < 0`，即便球队很强，也不一定值得买。

## 6. Hybrid model：基本面概率 + 赔率聚合

这个框架可以类比化工数字孪生中的 hybrid model：

```text
机理/基本面模型 + 数据/市场观测模型 -> 可交易概率判断
```

在世界杯盘口项目中，Goldman Sachs 报告相当于基本面模型输出。它不直接关心市场如何下注，而是基于球队实力、Elo、进攻火力、近期状态、地理因素、赛程路径等给出独立概率估计。

博彩公司赔率、交易所价格和 Polymarket 则相当于现场数据层。它们不是纯粹的真实概率，而是混合了公众情绪、流动性、庄家风控、热门球队溢价、新闻冲击和资金流。

项目真正研究的不是“哪个概率绝对正确”，而是二者残差：

```math
r_i = P_{GS,i} - P_{Market,i}
```

这个 residual 类似化工流程模拟中的“机理模型预测值 - 现场观测值”。偏差小，说明市场已经充分吸收基本面；偏差大，需要判断它是机会、噪声，还是模型遗漏。

对应关系可以写成：

```text
Goldman Sachs Elo-Poisson-MC 模型  ~=  反应器机理模型 / 流程模拟模型
赔率聚合与盘口水位                  ~=  DCS 实时数据 / 在线分析仪 / 软测量
基本面概率 - 市场隐含概率           ~=  模型预测值 - 现场观测值
下注决策                            ~=  优化控制 / 操作调整
```

因此，本项目更准确的定位不是“世界杯预测器”，而是“机构概率—市场价格”的 residual 分析系统。

## 7. 数据落地建议

Goldman Sachs 数据应拆成两层。

第一层是 `prior probabilities`，只保存报告明确给出的结果数据，例如夺冠概率、进各轮概率、关键路径概率。没有明确数字的不反推。

第二层是 `model metadata`，记录报告的模型结构和不可得数据，不伪装成原始数据库。

建议使用：

```text
fixtures/fundamental-priors/goldman_sachs_worldcup2026_prior.csv
fixtures/fundamental-priors/goldman_sachs_worldcup2026_metadata.yaml
```

市场侧数据仍然由现有 provider 与 pipeline 维护，例如 The Odds API、Polymarket、竞彩足球官方盘和其他补充源。最终核心指标来自两类数据合并后的比较。
